One-time setup (Cloudflare Pages — free)

Connect Cloudflare to GitHub once. No API token changes needed.

  1. Cloudflare dashboard → Workers & Pages → Create → Pages → Connect to Git
  2. Authorize GitHub → choose repo MonEx
  3. Settings:
    • Project name: monex-staging
    • Production branch: staging
    • Framework preset: None
    • Build command: bash scripts/prepare-staging-site.sh
    • Build output directory: staging-dist
  4. Save and Deploy

After the first build, bookmark your staging URL. If the Cloudflare project is named monex, the URL is https://monex.pages.dev/ (not monex-staging.pages.dev unless you name the project that way).

API (Cloudflare Worker)

Staging and live share the same API (monex-api). Game-only changes do not need an API redeploy.

On staging only, use the orange @test button (dev login as @test). The API allows this only from *.pages.dev origins (ENABLE_STAGING_DEV_AUTH=1). Keep global ENABLE_DEV_AUTH=0 on live — see Security go-live checklist.

Deploy API manually: Actions → Deploy Cloudflare API.
