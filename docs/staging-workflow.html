One-time setup (Cloudflare token)

Your GitHub secrets CLOUDFLARE_API_TOKEN and CLOUDFLARE_ACCOUNT_ID must allow Cloudflare Pages deploys.

  1. Cloudflare dashboard → My Profile → API Tokens
  2. Edit your deploy token (or create one) with:
    Account → Cloudflare Pages → Edit
    Account → Workers Scripts → Edit (you already use this for the API)
  3. Save the token in GitHub → repo Settings → Secrets → Actions

Then push any commit to staging (or run Deploy Staging (Cloudflare Pages) in Actions).

Daily workflow

StepWhat you do
1. DevelopCommit changes on branch staging (or merge your feature branch into staging)
2. Auto deployGitHub Actions deploys to Cloudflare Pages staging
3. TestOpen staging URL → orange STAGING banner at top confirms preview
4. Go liveGitHub → Pull requests → staging → main → Merge
5. Live siteGitHub Pages updates monexmonad.xyz (usually within a few minutes)

API (Cloudflare Worker)

Staging and live share the same API (monex-api). Game-only changes do not need an API redeploy.

Use dev login on staging. Deploy API manually when needed: Actions → Deploy Cloudflare API.

Branches

main     → live (GitHub Pages + monexmonad.xyz)
staging  → preview (Cloudflare Pages)

Keep staging up to date with main after each live release so the next test starts from production:

git checkout staging
git merge main
git push origin staging
